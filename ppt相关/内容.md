- 数据层：特征文件、参数数据库、GLB资产库

## 第4页 创新1：双重相对坐标 + 速度特征（1.5分钟）
### 页面标题
从134维到268维：让输入更稳、更懂动作

### 页面正文（可直接放PPT）
- 原始关键点（每帧134维）
  - Pose：25点 x 2 = 50维
  - 左手：21点 x 2 = 42维
  - 右手：21点 x 2 = 42维
- 双重相对化
  - Pose以鼻子为基准
  - 左手以左腕为基准
  - 右手以右腕为基准
- 速度差分
  - 对相对坐标做帧间差分，补充动态信息
- 最终输入
  - 相对坐标134维 + 速度134维 = 268维

### 可放公式
- \(\text{Feature}_{final} = [\text{Rel}(134),\ \Delta\text{Rel}(134)]\)
- \(134 = 50 + 42 + 42\)

### 结论点（加粗）
- 相对坐标解决“人站哪儿都能识别”
- 速度特征解决“静态像但动作不同”的区分问题

### 图示建议
- 两帧关键点叠加图：标注 Nose/Wrist 基准点与差分方向箭头

### 讲解词
如果直接喂绝对坐标，模型会被镜头位置和人站位干扰。我们把空间信息做了三路相对化，再加上速度场，让模型重点学习动作拓扑与运动趋势，而不是背景和拍摄条件。这一步是整个识别稳定性的基础。

### 过渡语
有了稳定输入，下一步是让模型抓住真正关键帧。

## 第5页 创新2：BiLSTM + Attention 时序建模（1.5分钟）
### 页面标题
不是看一帧，而是理解一段动作

### 页面正文（可直接放PPT）
- 输入张量：B x 64 x 268
- BiLSTM：双向建模前后文时序依赖
- Attention：对64帧分配不同权重，突出关键过渡帧
- 分类头：Dense + BN/LN + ReLU + Dropout -> 300类词汇概率

### 张量流（建议原样上屏）
- 原始序列：(T, 134)
- 特征增强：(T, 268)
- 重采样对齐：(64, 268)
- 模型输入：(B, 64, 268)
- BiLSTM输出：(B, 64, 512)
- Attention上下文：(B, 512)
- 分类输出：(B, 300)

### 结论点
- 手语语义主要存在于动作变化，不在单帧静态图像
- Attention 比“取最后一帧”更符合实际手语识别逻辑

### 图示建议
- 时间轴热度条：高权重帧对应动作转折点

### 讲解词
手语是典型时序信号。BiLSTM负责捕捉前后文，Attention负责告诉模型“哪几帧最关键”。这样做可以避免模型被冗余帧稀释信息，尤其对短词、高相似词的区分更有效。

### 过渡语
时序建模之外，线上稳定性还取决于左右手身份是否持续一致。

## 第6页 创新3：左右手鲁棒追踪与预处理（1分钟）
### 页面标题
复杂视频不“串手”：左右手身份保持一致

### 页面正文（可直接放PPT）
- 初始化策略：首帧按X坐标排序确定左右手
- 帧间追踪：最近邻匹配延续手身份
- 丢失容错：连续丢失阈值触发重置
- 轨迹修复：BBox插值补帧 + 高斯平滑（sigma=1.7）
- 采样策略：有效区间采样，统一到16~64帧

### 结论点
- 这不是“可选工程细节”，而是精度与稳定性的关键前提
- 没有身份一致性，模型会收到语义错位序列

### 图示建议
- 三帧示意：遮挡前 -> 遮挡中 -> 恢复后（ID保持）

### 讲解词
MediaPipe 在复杂场景下并不保证左右手标签稳定。我们加了最近邻追踪和重置机制，保证时间序列中的“左手还是左手、右手还是右手”。这一步显著降低了线上抖动和误识别。

### 过渡语
识别链路完成后，我们进入3D重建链路。

## 第7页 创新4：S2HAND三维重建与左手镜像（1.5分钟）
### 页面标题
只用右手模型，也能稳定重建双手

### 页面正文（可直接放PPT）
- 核心基座：S2HAND + MANO参数化手模型
- S2HAND输出：
  - joints: (21, 3)
  - vertices: (778, 3)
  - pose: 48维
  - shape: 10维
- 左手重建策略：
  1. 左手图像水平翻转后输入右手模型
  2. 推理输出X轴取反恢复左手坐标
  3. 面片绕序修正：faces[:, [0, 2, 1]]

### 结论点
- 核心是几何一致性恢复，不是简单翻图
- 该方案避免了单独训练左手模型，工程复用度高

### 图示建议
- 四步流程图：原图 -> 翻转输入 -> X回正 -> 面片修正

### 讲解词
S2HAND预训练主要是右手分布。我们设计了一套镜像恢复链：图像翻转、坐标反变换、面片法向修正，最终得到几何一致的左手网格。这样在不增加模型复杂度的情况下，实现了双手3D重建。

### 过渡语
得到手部网格后，还需要把它放回真实空间并保证观感自然。

## 第8页 创新5：空间还原与渲染优化（1分钟）
### 页面标题
从局部网格到可信动作：空间位移 + 动画平滑

### 页面正文（可直接放PPT）
- 问题：S2HAND输出是手腕局部坐标，不含全局位移
- 方案：利用 `meta.json` 的 offset/scale 恢复世界位移
- 空间映射：2D像素偏移 -> 3D世界坐标偏移
- 深度估计：基于“目标尺度与深度反比”近似恢复Z变化
- 渲染优化：
  - 顶点序列样条插值升频（x2）
  - 左右手独立帧计数，避免残影

### 关键公式（可放页角）
- \(V_{final} = (V_{local} + J_{root}) + Offset_{global}\)

### 讲解词
我们不是只导出“漂浮在原点的手网格”，而是通过元数据把动作放回真实运动轨迹中。再通过插值和平滑增强观感，最终输出可以直接播放的逐帧GLB动画序列。

## 第13页 答辩高频问答（备用页）
### Q1 为什么不用RGB端到端？
A：关键点方案更轻量，抗背景干扰更强，更适配边缘算力限制场景。

### Q2 为什么固定64帧？
A：在表达完整性和训练稳定性之间折中，便于批训练和实时推理。

### Q3 左手镜像会不会失真？
A：通过“图像翻转 + X轴回正 + 面片绕序修正”保证几何一致性。

### Q4 如何避免数据泄漏？
A：归一化均值方差仅由训练集统计，验证/测试不参与统计。

### Q5 工程可扩展性如何？
A：接口层、资源层、模型层解耦，后续扩词汇和扩功能只需局部改动。
