## 1. 项目简介与背景

### 1.1 这个项目是做什么的？

本项目是一个**手语识别后端服务**，将上传的视频转换为手语词汇预测，并返回与该词汇对应的 3D 动画资源地址。

你可以把它理解为一个闭环：

`视频输入 -> 骨架特征提取 -> 时序模型识别 -> 词条输出 -> 3D资源URL返回`

### 1.2 典型应用场景

| 场景 | 说明 |
|------|------|
| 手语翻译演示 | 用户上传视频后实时识别词语并显示结果 |
| 前后端联调 | 前端调用接口拿到 `word_name` 和 `.glb` 列表后进行播放 |
| 竞赛展示 | 展示完整工程链路：AI 推理 + 资源索引 + 静态文件服务 |

---

## 2. 解决的核心问题

### 问题一：如何将视频转成稳定可学习特征？

- 使用 MediaPipe Holistic 提取每帧关键点。
- 每帧固定输出 134 维：Pose(50) + Left Hand(42) + Right Hand(42)。
- 缺失关键点时补 0，保证维度稳定。

### 问题二：如何提升对时序动作的理解？

- 在 134 维基础上构建双重相对坐标与速度差分特征。
- 最终输入维度变为 268（相对坐标 134 + 速度 134）。
- 使用 BiLSTM + Attention 捕获前后文动作与关键帧权重。

### 问题三：如何把识别结果接到可展示资源？

- 启动时自动扫描 3D 模型目录并同步进 MySQL 表 `sign_assets`。
- 根据识别词条查库，返回按顺序排列的 `.glb` URL 列表。

## 6. 训练与推理流水线详解

### 6.1 阶段一：视频特征提取 `preprocess.py`

**目标**：从原始视频提取每帧 134 维关键点，并建立训练/验证/测试映射文件。

主要流程：

1. 读取 `nslt_300.json`，拿到视频分片区间与标签。
2. 对每个视频区间逐帧提特征，保存为 `.npy`。
3. 输出 `train_map_300.txt / val_map_300.txt / test_map_300.txt`。
4. 额外计算 134 维版本全局统计量（`global_mean_300.npy`, `global_std_300.npy`）。

### 6.2 阶段二：特征增强统计 `core_preprocess.py`

**目标**：将训练样本从 134 维扩展到 268 维，并生成训练用标准化参数。

核心函数：`to_double_relative_with_velocity(data)`

- 输入：`(T, 134)`
- 输出：`(T, 268)`
- 处理：
  - Pose 以鼻子点做相对化
  - 左手以左手腕做相对化
  - 右手以右手腕做相对化
  - 计算帧间差分速度
  - 拼接为 268 维

最终生成：

- `global_mean_300_double_vel.npy`
- `global_std_300_double_vel.npy`

### 6.3 阶段三：模型训练 `train.py`

**目标**：训练 300 类孤立词识别模型并保存最优权重。

流程：

1. 构建 `WLASLDataset` 与 `GeneratorDataset`。
2. 使用 `BiLSTMAttentionModel` 前向。
3. 损失函数：`CrossEntropyLoss`。
4. 优化器：`Adam`。
5. 验证集精度不提升时，学习率减半（手写 ReduceOnPlateau 逻辑）。
6. 保存 `best_model_300.ckpt` 和 `last_model_300.ckpt`。

### 6.4 阶段四：实时推理 `inference_camera.py`

**目标**：复用训练同构预处理，给序列输出词条与置信度。

关键模块：

- `LandmarkSmoother`：指数平滑，减少抖动。
- `extract_features(frame)`：单帧 134 维。
- `preprocess_sequence(raw_seq)`：`134 -> 268 -> 标准化 -> 64帧抽样`。
- `predict(raw_seq)`：模型前向 + Softmax + 标签映射。

设备策略：

- 优先尝试 Ascend。
- 不可用时自动回退 CPU。

### 6.5 阶段五：后端服务 `app.py`

**目标**：提供线上推理 API、资源索引 API 和静态资源服务。

启动动作：

1. 加载 `.env`。
2. 自动创建数据库与 `sign_assets` 表。
3. 扫描 `result_3d/glb_models` 并同步词条映射。
4. 初始化推理流水线 `SignLanguageInferencePipeline`。

### 6.6 核心代码片段与逐行详解

这一节补上你关心的“能讲清楚原理”的核心代码。以下片段均来自当前 backend 实现，并做了最小化整理，便于阅读。

#### 片段 A：模型架构（BiLSTM + Attention）

```python
class Attention(nn.Cell):
  def __init__(self, hidden_size):
    super(Attention, self).__init__()
    self.dense1 = nn.Dense(hidden_size * 2, hidden_size)
    self.tanh = nn.Tanh()
    self.dense2 = nn.Dense(hidden_size, 1)

  def construct(self, lstm_output):
    # lstm_output: [B, Seq, Hidden*2]
    scores = self.dense2(self.tanh(self.dense1(lstm_output)))  # [B, Seq, 1]
    weights = ops.softmax(scores, axis=1)
    context = ops.reduce_sum(weights * lstm_output, axis=1)    # [B, Hidden*2]
    return context


class BiLSTMAttentionModel(nn.Cell):
  def __init__(self, input_size, hidden_size, num_classes, num_layers=2, dropout=0.3):
    super(BiLSTMAttentionModel, self).__init__()
    self.lstm = nn.LSTM(
      input_size=input_size,
      hidden_size=hidden_size,
      num_layers=num_layers,
      batch_first=True,
      bidirectional=True,
      dropout=float(dropout) if num_layers > 1 else 0.0
    )
    self.ln = nn.LayerNorm((hidden_size * 2,))
    self.attention = Attention(hidden_size)
    self.fc1 = nn.Dense(hidden_size * 2, hidden_size)
    self.bn1 = nn.BatchNorm1d(hidden_size)
    self.relu = nn.ReLU()
    self.dropout = nn.Dropout(p=dropout)
    self.fc2 = nn.Dense(hidden_size, num_classes)

  def construct(self, x):
    # x: [B, 64, 268]
    out, (_, _) = self.lstm(x)     # [B, 64, Hidden*2]
    out = self.ln(out)
    context = self.attention(out)  # [B, Hidden*2]
    out = self.fc1(context)
    out = self.bn1(out)
    out = self.relu(out)
    out = self.dropout(out)
    out = self.fc2(out)            # [B, 300]
    return out
```

讲解要点：

1. BiLSTM 输出是双向拼接，所以是 `Hidden*2`。
2. Attention 不是直接取最后一帧，而是对全部时间步做加权求和，能突出关键动作帧。
3. 分类头使用 `Dense + BN + ReLU + Dropout + Dense`，用于增强表达并抑制过拟合。

#### 片段 B：训练循环（损失、反向传播、验证、学习率衰减）

```python
loss_fn   = nn.CrossEntropyLoss()
optimizer = nn.Adam(model.trainable_params(), learning_rate=config.LEARNING_RATE)

class WithLossCell(nn.Cell):
  def __init__(self, backbone, loss_fn):
    super().__init__()
    self.backbone = backbone
    self.loss_fn = loss_fn

  def construct(self, data, label):
    logits = self.backbone(data)
    return self.loss_fn(logits, label)

net_with_loss = WithLossCell(model, loss_fn)
train_cell = nn.TrainOneStepCell(net_with_loss, optimizer)

best_val_acc = -1.0
lr_patience, lr_wait = 5, 0
current_lr = config.LEARNING_RATE

for epoch in range(EPOCHS):
  train_cell.set_train(True)
  for batch in train_loader.create_dict_iterator():
    data = ops.cast(batch["data"], ms.float32)
    label = ops.cast(batch["label"], ms.int32)
    loss = train_cell(data, label)  # 前向 + 反向 + 参数更新

  val_acc = evaluate(model, val_loader)

  if val_acc > best_val_acc:
    best_val_acc = val_acc
    lr_wait = 0
    save_checkpoint(model, config.BEST_MODEL_PATH)
  else:
    lr_wait += 1
    if lr_wait >= lr_patience:
      current_lr *= 0.5
      lr_wait = 0
      optimizer = nn.Adam(model.trainable_params(), learning_rate=current_lr)
      train_cell = nn.TrainOneStepCell(net_with_loss, optimizer)

  save_checkpoint(model, config.LAST_MODEL_PATH)
```

讲解要点：

1. `TrainOneStepCell` 在 MindSpore 中把反向传播和参数更新封装成一步。
2. `best_model` 保存最优验证精度，`last_model` 保存最后一次状态，便于恢复训练。
3. 学习率策略是手工版 Reduce-on-Plateau：验证集连续不提升则减半。

#### 片段 C：核心特征工程（134 -> 268）

```python
def to_double_relative_with_velocity(data):
  # data: (T, 134)
  T = data.shape[0]

  pose = data[:, 0:50].reshape(T, 25, 2)
  lh   = data[:, 50:92].reshape(T, 21, 2)
  rh   = data[:, 92:134].reshape(T, 21, 2)

  nose = pose[:, 0:1, :]
  l_wrist = lh[:, 0:1, :]
  r_wrist = rh[:, 0:1, :]

  pose_rel = pose - nose
  lh_rel = lh - l_wrist
  rh_rel = rh - r_wrist

  pose_d = np.diff(pose_rel, axis=0)
  lh_d   = np.diff(lh_rel, axis=0)
  rh_d   = np.diff(rh_rel, axis=0)

  pose_d = np.concatenate([np.zeros_like(pose_d[:1]), pose_d], axis=0)
  lh_d   = np.concatenate([np.zeros_like(lh_d[:1]), lh_d], axis=0)
  rh_d   = np.concatenate([np.zeros_like(rh_d[:1]), rh_d], axis=0)

  final_feat = np.concatenate([
    pose_rel.reshape(T, -1), lh_rel.reshape(T, -1), rh_rel.reshape(T, -1),
    pose_d.reshape(T, -1), lh_d.reshape(T, -1), rh_d.reshape(T, -1)
  ], axis=1)

  return final_feat  # (T, 268)
```

讲解要点：

1. 相对坐标去掉了人物在画面中整体平移的影响。
2. 差分速度补足了“动作方向与快慢”信息。
3. 第一帧速度置零，确保时间长度与原序列一致。

#### 片段 D：推理预处理（标准化 + 64帧对齐）

```python
def preprocess_sequence(self, raw_seq):
  if len(raw_seq) < 5:
    return None

  raw_seq = np.array(raw_seq, dtype=np.float32)
  data = to_double_relative_with_velocity(raw_seq)   # (T, 268)

  if self.mean is not None:
    data = (data - self.mean) / self.std

  idx = np.linspace(0, len(data) - 1, self.seq_len)
  idx = np.round(idx).astype(int)
  data = data[idx]                                    # (64, 268)

  return data
```

讲解要点：

1. 训练和推理必须使用同一套 `mean/std`，否则分布漂移会显著拉低准确率。
2. 使用线性均匀抽样对齐到 64 帧，保证模型输入张量固定。

#### 片段 E：接口主流程（/api/sign/predict）

```python
@app.route('/api/sign/predict', methods=['POST'])
def predict():
  if 'video' not in request.files:
    return jsonify({"code": 400, "message": "Missing 'video' field"}), 400

  video_file = request.files['video']
  tmp = tempfile.NamedTemporaryFile(delete=False, suffix='.mp4')
  video_file.save(tmp.name)

  cap = cv2.VideoCapture(tmp.name)
  frame_buffer = []
  pipeline.smoother.prev_landmarks = None

  while True:
    ret, frame = cap.read()
    if not ret:
      break
    raw_feat = pipeline.extract_features(frame)
    smooth_feat = pipeline.smoother.smooth(raw_feat)
    if smooth_feat is not None:
      frame_buffer.append(smooth_feat)

  if len(frame_buffer) < 5:
    return jsonify({"code": 400, "message": "Video too short"}), 400

  word_name, confidence = pipeline.predict(frame_buffer)
  return jsonify({
    "code": 200,
    "data": {"word_name": word_name.lower(), "confidence": round(float(confidence), 4)}
  })
```

讲解要点：

1. 后端不会直接吃整段视频 tensor，而是逐帧提特征后再喂模型。
2. 先平滑再入缓冲，能减少关键点抖动导致的误判。
3. 返回统一 JSON 协议，便于前端稳定联调。

#### 片段 F：答辩时可以直接讲的张量尺寸变化

```text
单帧关键点:      (134,)
视频原始序列:    (T, 134)
双相对+速度:     (T, 268)
标准化+重采样:   (64, 268)
模型输入:        (B, 64, 268)
BiLSTM输出:      (B, 64, 512)    # hidden=256, 双向
Attention上下文: (B, 512)
分类输出logits:  (B, 300)
```

这组尺寸是评委最常问的“你这个网络每层到底在算什么”，建议背熟。

## 8. 核心设计决策解析（12条）

1. 采用关键点而非 RGB 端到端，降低背景干扰与计算负担。
2. 每帧固定 134 维，缺失补零，保证推理稳定。
3. 双重相对坐标消除全局平移影响。
4. 速度差分增强动作动态表达。
5. 统一为 64 帧，便于 batch 训练。
6. 训练集才做增强（缩放+噪声），验证/测试不增强。
7. 归一化统计量采用训练集生成，避免数据泄漏。
8. 模型结构选 BiLSTM + Attention，兼顾上下文和关键帧。
9. 优先 Ascend，自动回退 CPU，提高部署兼容性。
10. 服务启动即自动建表与资源同步，降低运维门槛。
11. 词条索引存入 MySQL，便于后续扩展多资源类型。
12. 前后端分离，后端统一输出 URL，前端只负责展示。

## 10. 接口规范与返回格式

### 10.1 `POST /api/sign/predict`

请求：

- `Content-Type: multipart/form-data`
- 字段：`video`

成功返回示例：

```json
{
  "code": 200,
  "data": {

    "confidence": 0.9134
  }
```

错误返回示例：

```json

  "code": 400,
  "message": "Missing 'video' field"
}
```

### 10.2 `GET /api/sign/downloads?name=HELLO`

成功返回示例：

```json
{
  "code": 200,
  "data": {
    "urls": [
      "http://127.0.0.1:5000/result_3d/glb_models/HELLO_xxx/frame_000.glb"

  }

```

未找到返回：

```json

  "code": 404,
  "msg": "Resource not found"
```

- 由 `send_from_directory` 提供文件下载/流式返回。

## 13. 常见问题与排错

### 问题 1：服务启动即退出

- 多数是数据库连接失败。
- 检查 `.env` 中账号密码、端口、数据库权限。

### 问题 2：`predict` 返回视频过短

- 当前要求至少 5 帧有效特征。
- 上传更长、动作更完整的视频。

### 问题 3：识别结果明显异常

- 检查 `global_mean_300_double_vel.npy` 与 `global_std_300_double_vel.npy` 是否存在。
- 检查 `best_model_300.ckpt` 与 `idx2name_300.txt` 是否匹配。

### 问题 4：`downloads` 返回 404

- 数据库中没有该词条映射。
- 检查 `result_3d/glb_models` 下是否存在对应目录。
- 重启后端触发自动同步。

### 问题 5：MindSpore 设备不匹配

- 非 Ascend 环境回退 CPU 属于正常行为。
- 只是速度差异，不影响功能正确性。
