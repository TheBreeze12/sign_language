#### 关键点索引结构

21 个关键点按如下标准排列（基于 MediaPipe 手部标签系统）：

| 索引 | 名称 | 位置 |
|------|------|------|
| 0 | WRIST | 手腕 |
| 1-4 | THUMB_MCP...IP | 大拇指 4 个关节 |
| 5-8 | INDEX_FINGER_MCP...IP | 食指 4 个关节 |
| 9-12 | MIDDLE_FINGER_MCP...IP | 中指 4 个关节 |
| 13-16 | RING_FINGER_MCP...IP | 无名指 4 个关节 |
| 17-20 | PINKY_MCP...IP | 小指 4 个关节 |

这意味着每根手指有 **MCP（掌指关节）→ PIP（近端指间关节）→ DIP（远端指间关节）→ TIP（指尖）** 四个监测点。

## 第二部分：关键点到骨骼动画的转换流程

### 2.1 总体处理流程图

```
原始关键点数据（21个点）
    ↓
[步骤1] 卡尔曼滤波（去噪）
    ↓

    ├─→ 批量更新 21 个球体位置
    ↓

    ↓
```

### 2.3 步骤 2：手掌姿态计算（核心算法）

这是整个骨骼驱动中最关键的部分，决定手掌的朝向。

#### 数学原理

**输入**：
- 点 0（手腕）：$P_0 = (x_0, y_0, z_0)$
- 点 5（食指根）：$P_5 = (x_5, y_5, z_5)$
- 点 9（中指根）：$P_9 = (x_9, y_9, z_9)$

**目标**：计算手掌局部坐标系的基向量

#### 算法步骤（代码位置：[Test.cs](Test.cs#L671-L690)）

```csharp
// 步骤 2.1：向量化
Vector3 wristPosition = new Vector3(hand.landmarks[0].x, hand.landmarks[0].y, hand.landmarks[0].z * 4);
Vector3 indexPosition = new Vector3(hand.landmarks[5].x, hand.landmarks[5].y, hand.landmarks[5].z * 4);
Vector3 middlePosition = new Vector3(hand.landmarks[9].x, hand.landmarks[9].y, hand.landmarks[9].z * 4);

Vector3 vectorToMiddle = middlePosition - wristPosition;  // 中指方向
Vector3 vectorToIndex = indexPosition - wristPosition;    // 食指方向
```

**几何意义**：
- `vectorToMiddle`：从手腕指向中指，代表手掌内侧主方向
- `vectorToIndex`：从手腕指向食指，代表手掌侧向

```csharp
// 步骤 2.2：正交归一化（Gram-Schmidt 正交化）
Vector3.OrthoNormalize(ref vectorToMiddle, ref vectorToIndex);
```

**作用**：
这个函数将两个向量转换为正交单位向量，防止由于关键点检测误差导致的向量不垂直。

**内部原理**：
```
1. 归一化 vectorToMiddle：v1 = vectorToMiddle / |vectorToMiddle|
2. 投影移除：temp = vectorToIndex - (vectorToIndex · v1) * v1
3. 归一化投影结果：v2 = temp / |temp|
结果：v1 ⊥ v2，均为单位向量
```

```csharp
// 步骤 2.3：计算法向量（右手坐标系）
Vector3 normalVector = Vector3.Cross(vectorToIndex, vectorToMiddle);
```

**矢量叉积公式**：
$$\vec{n} = \vec{v_1} \times \vec{v_2} = \begin{vmatrix} \vec{i} & \vec{j} & \vec{k} \\ v_1^x & v_1^y & v_1^z \\ v_2^x & v_2^y & v_2^z \end{vmatrix}$$

这给出垂直于手掌平面的方向（手背朝向）。

```csharp
// 步骤 2.4：使用 LookRotation 生成四元数
left_points[0].transform.localRotation = 
    Quaternion.LookRotation(normalVector, vectorToIndex) * Quaternion.Euler(0, 90, 90);
```

**LookRotation 的含义**：
- 第一参数（forward）：设定骨骼Z轴的方向（看向 normalVector）
- 第二参数（upwards）：约束Y轴方向（向上为 vectorToIndex）
- 因此X轴由右手法则自动确定

乘以 `Quaternion.Euler(0, 90, 90)` 是为了补正坐标系差异（从检测坐标系到骨骼模型局部坐标系）。

```csharp
// 步骤 2.5：镜像翻转（左右手处理不同）
left_points[0].transform.localRotation = Quaternion.Euler(
    left_points[0].transform.localRotation.eulerAngles.x,
    -left_points[0].transform.localRotation.eulerAngles.y,  // Y轴翻转
    -left_points[0].transform.localRotation.eulerAngles.z   // Z轴翻转
);
```

**原因**：左手和右手是镜像关系，需要按约定轴翻转以与动画模型对齐。

#### 代码完整示例（左手）

```csharp
// 【左手手掌姿态计算】
Vector3 wristPosition = new Vector3(hand.landmarks[0].x, hand.landmarks[0].y, hand.landmarks[0].z * 4);
Vector3 indexPosition = new Vector3(hand.landmarks[5].x, hand.landmarks[5].y, hand.landmarks[5].z * 4);
Vector3 middlePosition = new Vector3(hand.landmarks[9].x, hand.landmarks[9].y, hand.landmarks[9].z * 4);

Vector3 vectorToMiddle = middlePosition - wristPosition;
Vector3 vectorToIndex = indexPosition - wristPosition;

// 正交归一化
Vector3.OrthoNormalize(ref vectorToMiddle, ref vectorToIndex);

// 计算法向量（食指叉积中指 = 手背朝向）
Vector3 normalVector = Vector3.Cross(vectorToIndex, vectorToMiddle);

// 生成四元数并应用坐标系补正
left_points[0].transform.localRotation = 
    Quaternion.LookRotation(normalVector, vectorToIndex) * Quaternion.Euler(0, 90, 90);

// 镜像翻转
left_points[0].transform.localRotation = Quaternion.Euler(
    left_points[0].transform.localRotation.eulerAngles.x,
    -left_points[0].transform.localRotation.eulerAngles.y,
    -left_points[0].transform.localRotation.eulerAngles.z
);

// 【手腕位置】（带缩放与偏移）
HandLandmarkData mark0 = hand.landmarks[0];
left_points[0].transform.localPosition = 
    new Vector3(mark0.x / 2 - 0.15f, -mark0.y / 2 - 0.3f, mark0.z + hand.bound_area);
```

#### 右手的区别

右手使用 `Quaternion.Euler(0, 90, -90)` 而非 `(0, 90, 90)`，第三个参数符号不同，以适应右手的镜像坐标系。

简化位置计算也不同（+ 0.15f 变为 - 0.15f），使左右手在屏幕空间均匀分布。

### 2.4 步骤 3：逐指节骨骼驱动

#### 骨骼链的结构

一旦手掌基准点确定，剩余的 20 个关键点（5 根手指 × 4 个关节）需要驱动各自的**骨骼段**（bone）。

每根手指的 4 个点对应 4 个骨骼：
- 骨骼0：MCP 关节（掌指关节）
- 骨骼1：PIP 关节（近端指间关节）
- 骨骼2：DIP 关节（远端指间关节）
- ~~骨骼3~~：TIP 不作为独立骨骼，只作关键点

#### 驱动算法（代码位置：[Test.cs](Test.cs#L696-L710)）

```csharp
// 5根手指循环
for (int i = 0; i < 5; i++)
{
    // 每根手指的4个骨骼
    for (int j = 1; j <= 4; j++)
    {
        if (j < 4)  // 前三个骨骼
        {
            int index = i * 4 + j;  // 关键点索引 [1-19]
            
            // 获取相邻关键点的方向
            Vector3 direction = left_spheres[index + 1].transform.position 
                              - left_spheres[index].transform.position;
            
            // 骨骼 Forward 对齐到相邻两点的连线
            left_points[index].transform.forward = direction.normalized;
            
            // 保持原始的Z轴旋转（防止过度扭转）
            Quaternion currentRotation = left_points[index].transform.localRotation;
            Vector3 euler = currentRotation.eulerAngles;
            euler.z = origin_leftRotations[index].z;  // 恢复初始Z旋转
            left_points[index].transform.localRotation = Quaternion.Euler(euler);
        }
    }
}
```

#### 算法细节解析

1. **索引计算**：
   - 手指 i ∈ [0, 4]（5根手指）
   - 关节 j ∈ [1, 3]（3个要驱动的骨骼）
   - 关键点全局索引：`index = i * 4 + j` ⇒ 范围 [1, 19]

2. **方向向量**：
   ```
   direction = P_{index+1} - P_{index}
   ```
   即从当前关节指向下一关节的向量。

3. **Forward 对齐**：
   ```csharp
   left_points[index].transform.forward = direction.normalized;
   ```
   这个操作将骨骼的 Z 轴（forward）旋转到与关键点连线方向一致。

4. **Z轴锁定**：
   ```csharp
   euler.z = origin_leftRotations[index].z;
   ```
   防止骨骼围绕其自身长轴过度旋转，保证动画的稳定性和物理合理性。

#### 这种做法的优势

- **简洁高效**：直接利用相邻点向量，无需复杂的逆向运动学（IK）
- **稳定可靠**：Z轴锁定防止不自然的扭转
- **视觉准确**：关键点球体位置与骨骼端点基本对齐

## 第三部分：时序平滑与鲁棒性机制

### 3.1 关键点：卡尔曼滤波详解

#### 为什么需要滤波？

原始手部检测结果存在以下问题：
1. **高频抖动**：帧间噪声导致位置跳动 ±0.02~0.1 单位
2. **掉帧**：网络延迟或遮挡导致间歇性缺失
3. **置信度不稳定**：同一关键点的可信度波动

#### 卡尔曼滤波的设计


**一维卡尔曼滤波核心公式**（见 [KalmanFilter.cs](KalmanFilter.cs)）：

给定：
- 过程模型噪声方差：$q$ (process noise)
- 测量噪声方差：$r$ (measurement noise)

递推预测步：
$$P_k^- = P_{k-1} + q$$

其中 $P$ 为估计误差协方差。

卡尔曼增益：
$$K_k = \frac{P_k^-}{P_k^- + r}$$

状态更新：
$$\hat{x}_k = \hat{x}_{k-1} + K_k(z_k - \hat{x}_{k-1})$$

协方差更新：
$$P_k = (1 - K_k) P_k^-$$

**参数含义**：
- $q$ 越大 → 对新测量更信任 → 滤波越弱，跟踪越快但抖动大
- $r$ 越大 → 对新测量信任度低 → 滤波越强，平滑但滞后

#### 代码实现

单维滤波器（[KalmanFilter.cs](KalmanFilter.cs#L18)）：

```csharp
public float Update(float measurement)
{
    // 预测阶段：累加过程噪声协方差
    p = p + q;
    
    // 计算卡尔曼增益
    k = p / (p + r);
    
    // 状态更新：加权融合新测量
    x = x + k * (measurement - x);
    
    // 协方差更新
    p = (1 - k) * p;
    
    return x;
}
```

三维滤波器包装（[Vector3KalmanFilter.cs](Vector3KalmanFilter.cs)）：

```csharp
public Vector3 Update(Vector3 measurement)
{
    // 对 x, y, z 各维独立应用卡尔曼滤波
    float filteredX = xFilter.Update(measurement.x);
    float filteredY = yFilter.Update(measurement.y);
    float filteredZ = zFilter.Update(measurement.z);
    
    return new Vector3(filteredX, filteredY, filteredZ);
}
```

#### 批处理应用

手部滤波器（[HandLandmarkFilter.cs](HandLandmarkFilter.cs#L24)）：

```csharp
public void FilterLeftHandLandmarks(HandLandmarkData[] leftHand)
{
    // 首次调用时初始化 21 个三维滤波器
    if (!initialized && leftHand.Length >= LandmarkCount)
    {
        for (int i = 0; i < LandmarkCount; i++)
        {
            Vector3 leftPos = new Vector3(leftHand[i].x, leftHand[i].y, leftHand[i].z * 4);
            leftFilters[i].SetState(leftPos, 1.0f);  // 初始协方差 = 1.0
        }
        initialized = true;
    }
    
    // 每帧更新一次所有关键点
    for (int i = 0; i < Mathf.Min(LandmarkCount, leftHand.Length); i++)
    {
        Vector3 rawPosition = new Vector3(leftHand[i].x, leftHand[i].y, leftHand[i].z * 4);
        Vector3 filteredPosition = leftFilters[i].Update(rawPosition);
        
        // 写回原数组（就地修改）
        leftHand[i].x = filteredPosition.x;
        leftHand[i].y = filteredPosition.y;
        leftHand[i].z = filteredPosition.z / 4;  // 恢复原始 z 范围
    }
}
```

### 3.2 手势分类：滑动窗口多数表决

#### 为什么需要多数表决？

手势识别模型在每一帧都给出一个预测类别，但这些预测容易受网络传输抖动、光照变化等影响，导致**类别频繁跳变**（如在 "7" 和 "6" 之间抖动）。

多数表决是一种简单且有效的**离散信号平滑**方法。

#### 算法原理

**输入**：
- 窗口大小：`windowSize` = 5（可配置，见 [Test.unity](Test.unity#L1556)）
- 新识别手势：`newGesture`（如 "7"）
- 历史队列：`history`（存储最近 5 次结果）

**步骤**（代码位置：[Test.cs](Test.cs#L597)）：

```csharp
private string FilterGesture(Queue<string> history, string newGesture, int windowSize)
{
    // 1. 入队
    history.Enqueue(newGesture);
    
    // 2. 出队（保持窗口大小）
    while (history.Count > windowSize)
    {
        history.Dequeue();
    }
    
    // 3. 计数
    Dictionary<string, int> gestureCount = new Dictionary<string, int>();
    foreach (string gesture in history)
    {
        if (gestureCount.ContainsKey(gesture))
            gestureCount[gesture]++;
        else
            gestureCount[gesture] = 1;
    }
    
    // 4. 多数表决
    int maxCount = 0;
    string majorityGesture = newGesture;
    foreach (var pair in gestureCount)
    {
        if (pair.Value > maxCount)
        {
            maxCount = pair.Value;
            majorityGesture = pair.Key;
        }
        else if (pair.Value == maxCount)
        {
            // 并列时优先选择最新出现的
            foreach (string g in history.Reverse())
            {
                if (gestureCount[g] == maxCount)
                {
                    majorityGesture = g;
                    break;
                }
            }
        }
    }
    
    return majorityGesture;
}
```

#### 数学模型

假设窗口内的手势序列为：$[g_1, g_2, ..., g_w]$

定义计数函数：
$$C(l) = \sum_{i=1}^{w} \mathbb{1}(g_i = l)$$

其中 $\mathbb{1}$ 是示性函数，$l$ 为某一类别。

多数表决结果：
$$\hat{g} = \arg\max_l C(l)$$

若并列，选择最后出现的：
$$\hat{g} = \text{最后出现的最频繁类别}$$

#### 处理流程示例

假设 windowSize = 5，序列如下：

| 帧 | 原始识别 | 历史窗口 | 计数 | 结果 | 输出 |
|----|---------|---------|------|------|------|
| 1  | "7"     | ["7"]   | 7:1  | "7"  | 7    |
| 2  | "6"     | ["7","6"] | 7:1, 6:1 | "6" (最新) | 6 |
| 3  | "7"     | ["7","6","7"] | 7:2, 6:1 | "7" | 7 |
| 4  | "6"     | ["7","6","7","6"] | 7:2, 6:2 | "6" (最新) | 6 |
| 5  | "7"     | ["7","6","7","6","7"] | 7:3, 6:2 | "7" | 7 |
| 6  | "7"     | ["6","7","6","7","7"] (去掉最老的"7") | 7:3, 6:2 | "7" | 7 |
| 7  | "7"     | ["7","6","7","7","7"] | 7:4, 6:1 | "7" | 7 |

**效果**：
- 原始序列在 "7" 和 "6" 频繁抖动
- 滤波后在第 3 帧稳定为 "7"，除非连续的新测量打破稳定

#### 参数调优

- **windowSize = 3**：响应快，但对噪声不鲁棒
- **windowSize = 5~8**：平衡响应和平滑性（推荐）
- **windowSize = 10+**：高度平滑，但交互滞后

场景中配置（见 [Test.unity](Test.unity#L1556)）：
```
gestureWindowSize: 5
```

#### 实际应用

多数表决的结果用于**交互触发**（代码位置：[Test.cs](Test.cs#L791)）：

```csharp
if (filteredLeftGesture.Equals("7") && is_left_detected)
{
    left_hold.hold = true;  // 只有稳定识别为"7"时才触发抓取
}
```

## 第四部分：整体信号处理链

### 完整数据流

```
服务端推理结果（JSON）
    ↓ 网络接收（4字节长度头 + 数据）
    ↓ JsonUtility.FromJson（反序列化）
    ↓ GestureData 对象
    │
    ├─→ [连续信号处理]
    │   hand.landmarks[0..20]
    │   ↓ 卡尔曼滤波（3D）
    │   ↓ 坐标缩放变换
    │   ↓ 更新 21 个球体位置
    │   ↓ 计算手掌姿态（三点法）
    │   ↓ 驱动 20 个骨骼段方向
    │
    └─→ [离散信号处理]
        hand.hand_gesture（字符串）
        ↓ 滑动窗口多数表决
        ↓ 手势稳定输出
        ↓ 交互触发（如抓取）

最终结果
├─ 3D 骨骼动画（实时）
├─ 稳定手势识别
└─ 场景交互（放大、旋转、缩放）
```

## 第五部分：设计亮点与扩展方向

### 设计亮点

1. **双层平滑机制**
   - 连续信号（关键点）→ 卡尔曼滤波
   - 离散信号（手势）→ 多数表决
   - 分别优化，不会互相干扰

2. **几何驱动而非纯贴点**
   - 不是简单地把关键点投影到屏幕
   - 而是通过几何关系（向量、法向量、四元数）驱动骨骼链
   - 结果更符合物理约束和视觉预期

3. **主线程调度设计**
   - 网络 I/O 在子线程，不阻塞渲染
   - 通过 UnityMainThreadDispatcher 安全地回到主线程
   - 避免了竞态条件和死锁

### 潜在改进方向

1. **IK（逆向运动学）约束**
   - 当前是 FK（前向动画），直接驱动各关节
   - 可加入 IK 使得手指末端更精确对齐关键点
   - 权衡计算成本 vs 精度

2. **自适应滤波参数**
   - 当前 $q, r$ 固定
   - 可根据检测置信度动态调整
   - 或根据速度变化调整（快速运动时减弱滤波）

3. **关键点置信度**
   - 服务端通常返回置信度（如 0~1）
   - 可在数据结构中添加 `float confidence` 字段
   - 低置信度点使用更强的滤波或特殊处理

4. **手势组合识别**
   - 当前只基于当前帧的手势标签
   - 可考虑帧序列进行动作识别（如"手指向上3秒"）
   - 需要更复杂的状态机或时序模型

5. **多模型融合**
   - 同时运行 CPU 和 GPU 端的轻量级检测
   - 结果加权平均以提减低开销版本
   - 提高鲁棒性和实时性的平衡
