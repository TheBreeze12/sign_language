# 手势项目完整解读

## 1. 项目定位与整体目标
这个项目是一个基于 Unity 的实时手势交互系统，核心目标是把设备摄像头画面中的手部动作，实时转成 3D 手部骨骼动画，并将识别结果用于场景交互（抓取、旋转、缩放目标物体）。

从现有代码看，它不是“本地端到端推理”，而是“前端采集 + 网络推理 + 前端渲染交互”的架构：
- Unity 端负责采集帧、发送网络请求、接收识别结果、驱动 3D 手模型
- 服务器端负责手势识别与关键点推理（本仓库未包含服务端实现）

## 2. 项目能干什么

### 2.1 实时采集并上传相机帧
- 请求 Android 摄像头权限，并在授权后初始化相机
- 以固定帧间隔采集图像，压缩为 JPG，送入发送队列
- 通过 TCP 套接字发送到远端推理服务

对应实现：
- 相机与权限流程在 [Test.cs](Test.cs#L145)、[Test.cs](Test.cs#L162)
- 帧采集与压缩在 [Test.cs](Test.cs#L220)
- 网络连接与发送在线程中执行 [Test.cs](Test.cs#L313)、[Test.cs](Test.cs#L352)

### 2.2 实时接收手部识别结果
- 接收服务器回传的 JSON 数据
- 反序列化为结构化手部数据（手数、左右手、21 关键点、手势类别、边界面积）

对应实现：

- 接收与反序列化在 [Test.cs](Test.cs#L416)、[Test.cs](Test.cs#L480)

### 2.3 驱动双手 3D 骨骼动画
- 对左右手关键点分别做滤波
- 用关键点驱动 21 个可视化球体
- 根据手腕、食指根、中指根三点计算手掌姿态
- 逐指节更新骨骼朝向，实现手部动作同步

对应实现：

### 2.4 手势驱动场景交互
- 通过手势值（代码中为字符串 "7"）触发 hold 状态
- 单手 hold 时可持续传递位移与旋转

## 4. 架构与数据流（端到端）
1. Unity 每隔约 33ms 捕获一帧（约 30FPS）
2. 相机 YUV 纹理通过 Compute Shader 转换为 RGB
3. 帧编码为 JPG，进入线程安全队列
4. 发送线程将图像推送到 TCP 服务端
5. 接收线程读取服务端回传 JSON（手势 + 关键点）
6. 通过主线程调度器切回 Unity 主线程
7. 执行关键点滤波 + 骨骼更新 + 手势交互控制

## 5. 这个项目的创新点

### 5.1 实时链路设计比较完整
不是只做“识别演示”，而是把采集、传输、解析、渲染、交互串成完整闭环，并处理了线程切换与资源释放。

### 5.2 双层平滑机制
- 连续空间信号（关键点）用卡尔曼滤波
- 离散分类信号（手势类别）用多数投票

这种组合能明显减少“位置抖动 + 类别跳变”同时出现的问题。

### 5.3 几何驱动骨骼而非纯贴点
通过关键点方向关系推导手掌法向与指骨朝向，而不是只显示关键点云，交互感更自然。

### 5.4 交互策略可落地
手势值映射为抓取状态，再扩展为目标平移/旋转/双手缩放，具备直接用于 AR 交互原型的价值。

## 6. 当前实现边界与注意事项

### 6.1 服务端不在此仓库
本项目依赖外部识别服务，只有客户端代码。运行前必须准备兼容协议的 TCP 推理服务。

### 6.2 运行环境偏向 Android/XREAL
代码包含 Android 摄像头权限和 XREAL 相机组件依赖，PC 编辑器直接运行可能缺少输入源。

### 6.3 资源与工程完整性
当前工作区展示的是场景与脚本核心资产；若缺少完整 Unity 工程配置（ProjectSettings、Packages），需要在原工程中打开或补齐依赖包。

## 7. 怎么运行这个项目（实操步骤）

### 7.1 准备环境
1. 安装 Unity Editor（建议与原项目一致版本，若未知先尝试 2021/2022 LTS）。
2. 准备 Android Build Support（若目标设备是 Android/XREAL）。
3. 导入 XREAL 相关 SDK/示例包，确保 RGBCameraExample 可用。

### 7.2 准备推理服务
1. 启动手势识别服务端。
2. 确认协议：接收 JPG 帧并回传 JSON，JSON 字段需匹配 HandLandmarkData/HandData/GestureData。
3. 在场景或 Inspector 中配置服务器地址端口（默认见 [Test.unity](Test.unity#L1557)、[Test.unity](Test.unity#L1558)）。

### 7.3 检查场景对象绑定
1. 打开场景 [Test.unity](Test.unity)。
2. 确认挂载 Test 脚本的对象存在（场景中有绑定记录，见 [Test.unity](Test.unity#L1464)）。
3. 在 Inspector 里核对这些引用是否已赋值：
   - 左右手骨骼点列表 left_points/right_points
   - 关键点球体列表 left_spheres/right_spheres
   - RGBCameraExample、RawImage、RenderTexture、YUV2RGBShader
   - 左右 Hold 对象、目标对象、结果文本

### 7.4 运行与验证
1. 连接目标设备并授予相机权限。
2. 进入 Play 模式或打包到设备运行。
3. 观察是否出现：
   - 服务端连通
   - 关键点与手骨骼随手移动
   - UI 文本显示稳定手势结果
   - 单手抓取旋转、双手抓取缩放生效

### 7.5 常见问题定位
- 无画面/无关键点：优先检查相机权限、XREAL 输入源和 YUV2RGBShader 引用。
- 无手势结果：检查服务端是否启动、IP/端口是否可达。
- 抖动明显：调小 processNoise 或调大 measurementNoise，或增大手势窗口。
- 交互不触发：检查手势标签是否确实返回 "7"，以及 Hold 触发器碰撞配置。

## 8. 结论
这个项目已经具备一个可演示、可扩展的“实时手势到 3D 交互”原型系统：
- 工程价值在于链路完整和交互闭环
- 技术亮点在于多线程主线程协作 + 双层平滑 + 几何骨骼驱动
- 若补齐服务端和工程依赖，可直接用于 AR 手势交互 Demo、教学实验或比赛展示
